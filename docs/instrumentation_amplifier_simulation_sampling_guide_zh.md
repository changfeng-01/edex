# 给三运放仿真负责人的说明：为什么既要随机探索，也要成对扰动

## 先说结论

随机取值没有错，我们确实需要随机样本来观察整个参数空间，也需要用随机样本检验算法是否真的比随机搜索更好。

但如果所有参数都一起随机变化，我们只能知道“这一组结果好或不好”，很难判断到底是哪一个电阻或电容造成的，也无法可靠地把其他电路学到的改进方向转换成这个三运放电路的具体元件调整。

因此我们希望把仿真分成三类，而不是只做一种：

1. **全局探索样本**：覆盖整个参数范围；
2. **局部成对扰动样本**：判断每个参数的影响方向和大小；
3. **算法与随机盲测样本**：公平判断算法有没有价值。

三类样本解决的问题不同，不能互相替代。

## 为什么不能只做普通随机取值

假设某次仿真同时改变了 `R`、`RG`、`RD` 和 `CF`，最后发现带宽提高了。我们无法确定：

- 是 `CF` 变小带来的；
- 是 `RG` 变化带来的；
- 是多个参数共同作用；
- 还是某个参数改善带宽、另一个参数又抵消了一部分。

普通随机数还容易在某些区域堆在一起，在另一些区域完全没有样本。仿真次数有限时，这会浪费计算量。

所以全局探索仍然采用“随机思想”，但建议使用 Sobol 或拉丁超立方抽样。它们比简单地调用随机数更加均匀，并不会偏向算法喜欢的区域。

## 为什么还要做成对扰动

成对扰动的意思是：选定一个已经能够正常工作的基准电路，每次只改变一个参数，而且分别向上和向下改变一次。

例如只考察 `CF`：

```text
基准：CF = 10.000 pF
向上：CF = 10.513 pF
向下：CF =  9.512 pF
```

其他元件和仿真条件全部保持不变。这样才能回答：

> CF 增大一点时，带宽、CMRR、输出裕量和功耗分别怎样变化？

正、负两次都做，是为了减少仿真数值误差，并检查电路在基准点附近是否近似平滑。如果向上和向下的变化完全不对称，算法就会知道这里不能放心地进行大步调整。

这13次仿真只用于物理标定：

```text
1个基准 + 6个参数 × 正负两个方向 = 13次
```

它们**不是随机搜索结果，也不能用来证明算法效果**。

## 建议的实际仿真顺序

### 第一批：32组全局探索

使用 Sobol 或拉丁超立方方法，在以下范围内生成32组参数：

| 参数 | 范围 | 抽样方式 |
|---|---:|---|
| `R` | 5–50 kΩ | 对数空间 |
| `RG` | 0.5–20 kΩ | 对数空间 |
| `RD` | 5–50 kΩ | 对数空间 |
| `KD_plus` | 0.8–2.0 | 线性空间 |
| `KD_minus` | 0.8–2.0 | 线性空间 |
| `CF` | 1 pF–1 nF | 对数空间 |

`KD_plus` 和 `KD_minus` 需要同时覆盖比例相等、小失配和较大失配的情况，因为二者失配会明显影响 CMRR。

这32组先统一使用 TT、25°C、±5 V，以及相同的输入和负载条件。

### 第二批：13组局部标定

从第一批中选择一个满足以下条件的设计作为基准：

- 仿真正常收敛；
- 增益基本接近目标；
- 输出没有削顶；
- CMRR、带宽和压摆率能够正常测量。

围绕这个基准生成13组局部成对扰动。不要在第一批仿真完成前，把模板参考值当成已经验证的最佳基准。

### 第三批：算法与随机盲测

第一批结果交给算法后：

- 算法推荐8组新参数；
- 另外独立生成8组新的 Sobol 随机参数；
- 16组使用完全相同的仿真设置；
- 仿真完成前，不把结果反馈给算法。

最后比较两组的：

- 仿真成功率；
- 满足全部约束的数量；
- 最佳结果；
- 平均结果；
- 找到合格设计需要的仿真次数。

这一步才用于判断算法是否优于随机搜索。

## 第一阶段工作量

| 用途 | 仿真数量 |
|---|---:|
| 全局探索 | 32 |
| 局部物理标定 | 13 |
| 算法推荐盲测 | 8 |
| 随机对照盲测 | 8 |
| 合计 | 61 |

第一阶段全部在标称工况运行。确认数据和算法流程有效后，再把基准、算法优选结果和随机对照中的代表性设计放到高温、低温、低电压及其他模型条件下验证，而不是一开始把所有设计都乘以全部 PVT 场景。

## 每次仿真需要返回什么

每一行参数对应一个唯一的 `run_id`。请返回：

- 仿真是否成功；
- 差模增益；
- 共模增益；
- CMRR；
- −3 dB 带宽；
- 输出摆幅或输出裕量；
- 输入共模裕量；
- 压摆率；
- 电源电流和功耗；
- 原始 AC 曲线、瞬态波形和仿真日志；
- 使用的运放模型、仿真器和版本。

仿真失败的样本也不要删除。失败本身就是参数边界的重要信息，需要保留 `run_id` 和失败原因。

## 我们真正希望避免的情况

我们不是反对随机取值，而是希望避免：

- 只随机取值，却无法判断单个参数的作用；
- 只做13组局部扰动，却声称算法整体有效；
- 算法推荐组和随机组使用不同仿真预算；
- 只返回“最好的一组”，丢失其他结果和失败样本；
- 只返回综合分数，没有原始电学指标和波形；
- 用算法已经看过的数据再次充当盲测数据。

## 一句话说明

> 随机样本负责看全局，成对扰动负责看清每个元件的作用，最后再用相同数量的算法推荐和随机样本做盲测。这样既保留随机探索，也能知道算法为什么这样改，并且能够公平判断算法到底有没有用。
